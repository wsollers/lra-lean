\begin{example*}
For Euclidean space, the Mathlib-first construction is to choose a finite index
type and use
\[
  \texttt{EuclideanSpace ℝ I}.
\]
For the Euclidean plane, we take \(I = \texttt{Fin 2}\). In Lean, the chapter
uses the alias
\[
\begin{gathered}
  \texttt{abbrev cartesianSpace : Type :=}\\
  \texttt{\ \ EuclideanSpace ℝ (Fin 2).}
\end{gathered}
\]
This is the correct Mathlib object when we want the usual Euclidean \(L^2\)
metric on \(\mathbb{R}^2\). We do not need to build a new metric-space
structure: Mathlib already supplies the metric-space instance.
\end{example*}

\begin{remark*}
The points \((0,0)\) and \((1,1)\) are represented by coordinate vectors and
then transported into \texttt{EuclideanSpace ℝ (Fin 2)}:
\[
\begin{gathered}
  \texttt{noncomputable def cartesianOrigin : cartesianSpace :=}\\
  \texttt{\ \ (EuclideanSpace.equiv (Fin 2) ℝ).symm ![0, 0]}\\
  \texttt{noncomputable def cartesianOneOne : cartesianSpace :=}\\
  \texttt{\ \ (EuclideanSpace.equiv (Fin 2) ℝ).symm ![1, 1].}
\end{gathered}
\]
The vector notation \texttt{![0, 0]} and \texttt{![1, 1]} gives functions out
of \texttt{Fin 2}. The equivalence \texttt{EuclideanSpace.equiv (Fin 2) ℝ}
relates those coordinate functions to Mathlib's Euclidean space; \texttt{.symm}
moves from coordinates into Euclidean space.
\end{remark*}

\begin{example*}
The distance from \((0,0)\) to \((1,1)\) is \(\sqrt{2}\):
\[
\begin{gathered}
  \texttt{lemma dist\_cartesianOrigin\_cartesianOneOne :}\\
  \texttt{\ \ dist cartesianOrigin cartesianOneOne = Real.sqrt 2 := by}\\
  \texttt{\ \ rw [EuclideanSpace.dist\_eq, Fin.sum\_univ\_two]}\\
  \texttt{\ \ simp [cartesianOrigin, cartesianOneOne,}\\
  \texttt{\ \ \ \ EuclideanSpace.equiv, Real.dist\_eq]}\\
  \texttt{\ \ ring\_nf.}
\end{gathered}
\]
The proof first rewrites Mathlib's Euclidean distance using
\texttt{EuclideanSpace.dist\_eq}. Since the index type is \texttt{Fin 2}, the
finite sum is rewritten as two coordinate terms by
\texttt{Fin.sum\_univ\_two}. The simplifier then evaluates the coordinates and
\texttt{ring\_nf} normalizes the arithmetic under the square root, leaving
\[
  \sqrt{(1-0)^2 + (1-0)^2} = \sqrt{2}.
\]
\end{example*}

\begin{remark*}
The file also keeps a scratch bridge example for the old coordinate formula on
\(\mathbb{R} \times \mathbb{R}\). The conversion
\[
  \texttt{toE : ℝ × ℝ -> EuclideanSpace ℝ (Fin 2)}
\]
turns an ordered pair into a Mathlib Euclidean point. The lemma
\[
  \texttt{distance\_eq\_euclidean}
\]
shows that the hand-written square-root formula agrees with Mathlib's
\texttt{dist} after this conversion. This is the bridge pattern: prove that a
scratch formula agrees with Mathlib's API, then use Mathlib theorems such as
\texttt{dist\_triangle} instead of reproving the coordinate triangle
inequality.
\end{remark*}
