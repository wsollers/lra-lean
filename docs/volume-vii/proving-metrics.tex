\begin{remark*}
The scratch metric files remain useful because they show what Mathlib's
\texttt{MetricSpace} typeclass is hiding from day-to-day proofs. A scratch
metric has the form
\[
\begin{gathered}
  \texttt{structure ScratchMetric (Point : Type u) where}\\
  \texttt{\ \ distance : Point -> Point -> Real}\\
  \texttt{\ \ distance\_nonnegative : forall x y : Point, 0 <= distance x y}\\
  \texttt{\ \ distance\_eq\_zero\_iff\_equal :}\\
  \texttt{\ \ \ \ forall x y : Point, distance x y = 0 <-> x = y}\\
  \texttt{\ \ distance\_symmetric : forall x y : Point, distance x y = distance y x}\\
  \texttt{\ \ distance\_triangle :}\\
  \texttt{\ \ \ \ forall x y z : Point, distance x z <= distance x y + distance y z.}
\end{gathered}
\]
Building a value of this structure means supplying the proposed distance
function and proving the metric laws by hand.
\end{remark*}

\begin{example*}
For the real line, the scratch metric is
\[
\begin{gathered}
  \texttt{def realScratchMetric : ScratchMetric Real where}\\
  \texttt{\ \ distance x y := |x - y|.}
\end{gathered}
\]
The proof obligations are the familiar facts about absolute value:
nonnegativity follows from \texttt{abs\_nonneg}; zero distance is equivalent to
equality by \texttt{abs\_eq\_zero} and elementary arithmetic; symmetry follows
from \texttt{abs\_sub\_comm}; and the triangle inequality follows from
\texttt{abs\_add\_le} after rewriting \(x-z\) as \((x-y)+(y-z)\).
\end{example*}

\begin{remark*}
The scratch discrete metric is kept as another proof-obligation exercise:
\[
\begin{gathered}
  \texttt{def discreteScratchMetric}\\
  \texttt{\ \ \ \ (Point : Type u) [DecidableEq Point] : ScratchMetric Point where}\\
  \texttt{\ \ distance x y := if x = y then 0 else 1.}
\end{gathered}
\]
The assumption \texttt{[DecidableEq Point]} lets Lean decide which branch of
the \texttt{if} expression applies. Proofs about this metric usually proceed by
\texttt{by\_cases h : x = y} followed by \texttt{simp [h]}.
\end{remark*}

\begin{remark*}
The scratch material is not the main development path. Once the raw obligations
are understood, the main theorems should be stated with Mathlib's context
\[
  \texttt{\{X : Type u\} [MetricSpace X]}.
\]
That shift lets us use Mathlib's API directly: \texttt{dist\_triangle} instead
of a local triangle field, \texttt{Metric.ball} instead of a custom ball
structure, and topological facts that come automatically from the metric-space
typeclass hierarchy.
\end{remark*}
