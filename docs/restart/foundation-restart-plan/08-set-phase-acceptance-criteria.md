# Step 8: Set Phase Acceptance Criteria

## Goal

Define the gate for moving beyond sets.

## Required Before Relations

- The set representation choice is documented.
- The first set core compiles.
- The proof roadmap lists a sequence of user-provable `sorry` statements.
- Each set declaration has a source-grounding note.
- Any proposed book tweaks are listed explicitly.
- The build command for the new foundation skeleton is documented and passes.

## Non-Goals

This gate does not require every set theorem to be fully proved. The point is
to make the architecture stable and proof-ready, not proof-complete.

## Exit Decision

Only after this gate is satisfied should the project consider relation
foundations.
